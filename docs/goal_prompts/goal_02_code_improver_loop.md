# Goal Prompt 02 — Code Improver Loop

## Goal

Implement the first recursive improvement loop: a local model proposes Python code changes, a sandbox runs tests, and the controller archives results and selects the best candidate.

## Required flow

```text
load task
→ load seed solution
→ call local model for candidate replacement
→ write candidate to temporary sandbox
→ run pytest with timeout
→ score candidate
→ append attempt to JSONL archive
→ repeat for N generations
→ report best candidate
```

## Task format

Each task directory should contain:

```text
prompt.md
seed_solution.py
tests.py
```

## Model interface

Implement a provider abstraction:

```python
class ModelClient:
    def generate(self, prompt: str) -> str: ...
```

Include a llama.cpp / LlamaBarn implementation (talk to its OpenAI-compatible API at `http://127.0.0.1:2276/v1`) but keep it optional/configurable. This minimal `ModelClient` is generalized into a full provider abstraction (local + Claude + GPT) in Goal 07 — keep the interface small and provider-neutral so that refactor is non-breaking.

## Evaluation

The evaluator should capture:

- return code
- stdout
- stderr
- runtime
- pass/fail
- number of tests passed if available

## Acceptance criteria

- A sample task can run for 5 generations.
- Each attempt is logged to `runs/attempts.jsonl`.
- The best candidate is selected by score.
- Candidate code runs in a temporary directory.
- Timeout is enforced.
- Tests cannot be modified by the candidate.

## Safety constraints

- Candidate execution has no network access.
- Candidate cannot alter evaluator code.
- Candidate cannot alter archive history.
- No shell commands are generated by the model directly; only the controller runs fixed commands.

## Self-improvement

This goal **is the inner loop** of the self-improvement cycle (`../13_self_improvement_loop.md`).

- **Records**: every candidate attempt — passing *and* failing — with its score and failure reason, into the archive. Negatives are first-class data, never discarded.
- **Reflects / proposes**: the controller reflects on archived scores and proposes the next candidate; selection keeps the best candidate as the seed for the next generation.
- **Validated / gated**: the sandbox runs the fixed test suite and the evaluator scores it; only a reproducible score improvement is selected.
- **Bounds**: per `../13_self_improvement_loop.md` — the model proposes text/patches only; the controller (not the model) runs fixed commands; no test/evaluator edits.

# Operational Pilot Plan

This is the fixed Goal 20 pilot plan. It measures whether frontier spend buys objective,
reproducible, safety-passing research progress over Tier 0. It is not a scale-up approval.

Generate the machine-readable plan and command transcript with:

```zsh
UV_CACHE_DIR=.uv-cache uv run siro pilot-init
```

The generated plan is written to `runs/pilots/operational-pilot-v1/pilot_plan.json`; the
expected report path is `runs/pilots/operational-pilot-v1/pilot_report.md`. `pilot-init`
also snapshots the exact tier/provider configs under `runs/pilots/operational-pilot-v1/configs/`.

## Benchmark Tasks

The fixed task set is:

- `packs/ml/tasks/algorithm/pair_count`
- `packs/ml/tasks/training/tiny_mlp`
- `packs/ml/tasks/policy/sentiment_rules`
- `packs/ml/tasks/data_cleaning/normalize_emails`
- `packs/ml/tasks/parser_validator/slug_validator`

Do not edit task evaluators, hidden data, gates, or promotion rules during a pilot. A change
to any of those inputs stops the current pilot and requires a new plan.

## Arms

Run every required arm against the same task set and archive each arm into its own directory:

- `tier0_local`: `config/tier0.local.yaml`, max spend `$0.00`.
- `tier1_cheap_frontier`: `config/tier1.cheap_frontier.yaml`, max spend `$2.00` per run and
  `$10.00` per day.
- `tier1_strong_frontier`: `config/tier1.frontier.yaml`, max spend `$5.00` per run and
  `$25.00` per day. Run this arm only if the cheap-frontier arm produces useful signal.

The command transcript generated by `pilot-init` is the authority for paths. Each arm writes:

- `research_attempts.jsonl`
- `model_calls.jsonl`
- config snapshots under `configs/`
- any human transcript notes needed to explain operator decisions

The direct wrapper is:

```zsh
UV_CACHE_DIR=.uv-cache uv run siro pilot-run --plan runs/pilots/operational-pilot-v1/pilot_plan.json
```

By default this runs required arms only. Use `--arm tier1_strong_frontier` or
`--include-conditional` only after the cheap-frontier arm produces useful signal.

## Stop Conditions

Stop and escalate if any of these happen:

- An arm exceeds its configured per-run or per-day spend cap.
- Any execution-plane network, package install, evaluator edit, safety-gate edit, hidden-data
  change, or benchmark-task change is requested.
- A provider binding, budget, task set, or promotion rule would change mid-pilot.
- The hard resource isolation backend is required for a target environment but is not ready.

## Report

Render the report with:

```zsh
UV_CACHE_DIR=.uv-cache uv run siro pilot-report
```

The report must include total cycles, estimated spend from `model_calls.jsonl`, provider
dashboard reconciliation when available, pass rate, promotion rate, hidden-test failure rate,
reproducibility failure rate, safety escalation rate, cost per accepted promotion, cost per
family, common failure signatures, and a `continue`, `revise`, or `stop` recommendation.

The recommendation is evidence-gated. No scale-up recommendation is valid unless the benchmark
task list, budgets, model-call ledger, attempt archive, and safety evidence are present.
